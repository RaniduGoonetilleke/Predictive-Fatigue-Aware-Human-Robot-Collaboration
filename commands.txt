# NETWORK SETUP

Robot IP:  192.168.1.10
Laptop IP: 192.168.1.100
Interface: eno1

Set laptop IP:
  sudo ip addr add 192.168.1.100/24 dev eno1
  sudo ip link set eno1 up

Test connection:
  ping 192.168.1.10


# BUILD 

Navigate to workspace:
  cd ~/ros2_ws

Build everything:
  colcon build

Build only your package:
  colcon build --packages-select vision_input

  source install/setup.bash

# USEFUL DEBUG COMMANDS
See all active topics:
  ros2 topic list

Watch gesture output:
  ros2 topic echo /vision_input/gesture_text

Watch joint states from robot:
  ros2 topic echo /joint_states

Watch what commands you're sending:
  ros2 topic echo /scaled_joint_trajectory_controller/joint_trajectory

Check if robot is publishing:
  ros2 topic hz /joint_states

Live tune mapping parameters:
  ros2 param set /robot_controller map_scale_y 0.8
  ros2 param set /robot_controller map_scale_z 0.8
  ros2 param set /robot_controller map_offset_z 0.25
  ros2 param set /robot_controller map_offset_x 0.30

List all parameters:
  ros2 param list /robot_controller


# MANUAL JOINT COMMAND (Testing without gestures)
Send robot to a position manually:
  ros2 topic pub --once /scaled_joint_trajectory_controller/joint_trajectory trajectory_msgs/msg/JointTrajectory "{
    header: {stamp: {sec: 0, nanosec: 0}},
    joint_names: [shoulder_pan_joint, shoulder_lift_joint, elbow_joint, wrist_1_joint, wrist_2_joint, wrist_3_joint],
    points: [{positions: [0.0, -1.57, 1.57, -1.57, -1.57, 0.0], velocities: [0,0,0,0,0,0], time_from_start: {sec: 4, nanosec: 0}}]
  }"


# RECORD JOINT ANGLES (For fetch pickup poses)
Jog robot to position using teach pendant, then:
  ros2 topic echo /joint_states --once

Copy the positions array — those are your pickup joints.

# SAFETY CHECKLIST (Before every session)
1. E-stop within reach
2. Speed slider at 10%
3. Launch system
4. Test STOP gesture first — verify freeze
5. Test OKAY gesture — verify unfreeze
6. Then test POINT
7. Increase speed gradually (25%, 50%)
